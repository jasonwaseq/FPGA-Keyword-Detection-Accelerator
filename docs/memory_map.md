# Memory Map and Data Layouts

## Physical memory budget (iCE40UP5K)

| Resource | Used | Total |
|---|---|---|
| EBR (4 Kbit block RAM) | 19 | 30 |
| SPRAM (32 KB single-port) | 0 | 4 |
| SB_MAC16 (DSP) | 7 | 8 |

## Feature buffer

* Geometry: `HIST_DEPTH=64` frames × `NUM_MFCC=40` INT8 coefficients.
* Address: `{frame[5:0], coef[5:0]}` — coefficient field padded to a
  64-slot power-of-two stride (24 unused bytes/frame, 3 extra EBR, zero
  address arithmetic). 4096×8 → 8 EBR.
* Write port: packet decoder (speculative bytes + commit pointer).
* Read port: conv engine, `frame = (win_base + t + k) mod 64` — the modulo
  is free (power-of-two depth).

## Weight / parameter images (generated by `model/kws_quant.py`)

All `$readmemh` images; paths are relative to the repository root and baked
at synthesis time. Regenerate → re-synthesize.

### `conv_weights.mem` — 960 × 8-bit
`index = (oc·K + k)·IC + ic` (output-channel major). Loaded into `PARALLEL`
identical ROM banks (2 EBR each); lane L computes output channels
`oc_group·P + L` and addresses its bank at `oc·120 + k·40 + ic`, maintained
incrementally.

### `dense_weights.mem` — 480 × 8-bit
`index = class·120 + (t·8 + ch)`. The flatten order `t·CH+ch` matches the
pooled RAM layout, so the classifier walks both memories with single
incrementing addresses. 1 EBR.

### `conv_bias.mem` — 10 × 32-bit, `dense_bias.mem` — 6 × 32-bit
`[bias[0..N-1], M, S]` per layer: biases in the INT32 accumulator domain
(bounded to ±2²³, asserted), then the layer's requantization multiplier and
shift. Synthesizes to LUT logic (a handful of constants).

## Activation RAMs

| RAM | Depth | Layout | Writer → Reader |
|---|---|---|---|
| Conv activations | 240×8 | `addr = t·8 + oc` | conv → pooling (1 EBR) |
| Pooled activations | 120×8 | `addr = p·8 + ch` | pooling → classifier (1 EBR) |

Time-major, channel-minor everywhere: pooling reads input rows contiguously
(`(p·POOL+q)·8 + ch`) and writes `p·8+ch`; the classifier's dense index *is*
the pooled address. No multiplier appears in any address path — all bases
update by constant increments.

## FIFOs

| FIFO | Depth | Overflow policy |
|---|---|---|
| RX (line → decoder) | 256 B (4.5 max-size packets) | drop newest + `fifo_overflows` counter (line can't be back-pressured) |
| TX (encoder → line) | 128 B | cannot overflow: encoder blocks on almost-full (level ≥ 126) |

## Statistics

* Live bank: 16 × 32-bit counter registers (map in [protocol.md](protocol.md)).
* Snapshot: 16 × 32-bit EBR (2 EBR), sequentially captured on `READ_STATS`,
  synchronously read by the response path.
